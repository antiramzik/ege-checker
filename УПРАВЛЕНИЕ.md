# Управление ботом (работает на GitHub Actions)

Бот живёт в репозитории **https://github.com/antiramzik/ege-checker** и запускается
**каждые 15 минут** на серверах GitHub (бесплатно). Когда появляется новый балл — пишет
в заданную Telegram-группу.

> Локальный запуск на Mac отключён (чтобы не было двойных уведомлений).

## Где смотреть

- **Запуски и логи:** https://github.com/antiramzik/ege-checker/actions
  Зелёная галочка = проверка прошла. Откройте любой запуск → шаг «Проверить результаты» → лог.

## Команды (по желанию, в Терминале)

**Проверить прямо сейчас, не ждать 15 минут:**
```
gh workflow run check.yml --repo antiramzik/ege-checker
```
(или на странице Actions → «Проверка результатов ЕГЭ» → Run workflow)

**Список последних запусков:**
```
gh run list --repo antiramzik/ege-checker --limit 5
```

**Остановить бота (выключить расписание):**
```
gh workflow disable "Проверка результатов ЕГЭ" --repo antiramzik/ege-checker
```

**Снова включить:**
```
gh workflow enable "Проверка результатов ЕГЭ" --repo antiramzik/ege-checker
```

## Важно

- **CapMonster платный** (~$0.0003/капча). Баланс ~$0.21 — примерно на неделю при проверке раз
  в 15 минут. Пополнить: capmonster.cloud.
- GitHub может запускать задачу с задержкой в несколько минут — это нормально.
- Если в репозитории 60 дней нет активности, GitHub отключает расписание (для пары недель — не проблема).
- Уже опубликованные результаты помечаются как виденные; уведомление приходит только на новые.
- Поменять данные/токены — в **Settings → Secrets and variables → Actions** репозитория.

## Вернуться к запуску на Mac (если понадобится)

```
mv ~/Library/LaunchAgents/com.ege.checker.plist.disabled ~/Library/LaunchAgents/com.ege.checker.plist
launchctl load -w ~/Library/LaunchAgents/com.ege.checker.plist
```
(тогда лучше выключить GitHub-расписание, чтобы не дублировать.)
