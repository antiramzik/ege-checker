# Бесплатный хостинг 24/7 + уведомления в Telegram-канал

Бот будет жить на **GitHub Actions** — это бесплатно навсегда для **публичного** репозитория
(карта не нужна). Раз в 15 минут GitHub сам запускает проверку и, если появились результаты,
бот пишет в твой Telegram-канал.

> Личные данные и ключи хранятся в «Secrets» — их **не видно** в публичном репозитории.
> Файл `config.json` загружать на GitHub НЕ нужно (он в `.gitignore`).
> В публичный репозиторий попадают только ID экзаменов (числа) — **баллы видны только в Telegram**.

---

## Шаг 1. Telegram: бот + канал

1. Открой **[@BotFather](https://t.me/BotFather)** → отправь `/newbot` → задай имя и username бота.
   BotFather выдаст **токен** вида `8123456789:AAH...` — это `TELEGRAM_BOT_TOKEN`.
2. Создай канал: Telegram → «Создать канал». Можно приватный.
3. Добавь своего бота в канал **администратором** (Управление каналом → Администраторы →
   Добавить → найди своего бота → дай право «Публикация сообщений»).
4. Узнай `TELEGRAM_CHAT_ID` канала — любым способом:
   - **Публичный канал:** chat_id = `@имя_канала` (его username). Самый простой вариант.
   - **Приватный канал:** добавь в канал бота **[@username_to_id_bot](https://t.me/username_to_id_bot)**
     (или перешли любой пост канала ему) — он покажет id вида `-1001234567890`. Это и есть chat_id.

---

## Шаг 2. GitHub: репозиторий

1. Заведи аккаунт на **github.com** (бесплатно).
2. **New repository** → имя любое (напр. `ege-checker`) → тип **Public** → Create.
   (Public важен: для публичных репозиториев минуты Actions бесплатны без лимита.)
3. Загрузи файлы проекта (кнопка **Add file → Upload files**), а именно:
   ```
   checker.py
   regions.json
   globalsign-intermediate.pem
   state.json
   requirements.txt
   .gitignore
   .github/workflows/check.yml
   ```
   ⚠️ **`config.json` НЕ загружай** (в нём личные данные).
   Папку `.github/workflows` проще создать так: Add file → Create new file →
   в имени введи `.github/workflows/check.yml` и вставь содержимое.

---

## Шаг 3. GitHub: секреты

Репозиторий → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**.
Добавь по одному:

| Имя секрета | Значение |
|---|---|
| `EGE_SURNAME` | Фамилия |
| `EGE_NAME` | Имя |
| `EGE_PATRONYMIC` | Отчество |
| `EGE_DOC_NUMBER` | номер документа без серии |
| `EGE_REGION` | название региона |
| `CAPMONSTER_KEY` | ключ из ЛК CapMonster |
| `TELEGRAM_BOT_TOKEN` | токен от BotFather |
| `TELEGRAM_CHAT_ID` | `@канал` или `-100…` |

> ⚠️ Не вставляй реальные данные в этот файл — только в Secrets репозитория (они не видны публично).

---

## Шаг 4. Запуск

1. Вкладка **Actions** → если просит — нажми «I understand… enable workflows».
2. Слева выбери **«Проверка результатов ЕГЭ»** → справа **Run workflow** (проверить прямо сейчас).
3. Дальше GitHub сам запускает проверку каждые 15 минут.
   Когда появится результат по предмету — придёт сообщение в канал. 🎉

Чтобы остановить — отключи workflow (Actions → … → Disable workflow) или удали репозиторий.

---

## Важно знать

- **CapMonster платный** (но дёшево: ~$0.0003 за капчу). Баланс на твоём ключе ~$0.21 —
  это примерно на неделю при проверке раз в 15 минут. Если ждать дольше — пополни на capmonster.cloud.
- GitHub может запускать задачу не ровно каждые 15 минут, а с задержкой в несколько минут — это норма.
- Если в репозитории 60 дней не было активности, GitHub выключает расписание (для пары недель ожидания — не проблема).
- Уже опубликованные результаты бот помечает как «виденные» и повторно о них не пишет —
  уведомление приходит только на новые.
