# Бот проверки результатов ЕГЭ

Сам заполняет данные участника, решает капчу (CapMonster или бесплатный OCR) и раз в N минут
проверяет на `checkege.rustest.ru`, появились ли результаты. Когда по предмету появляется
результат — уведомляет (Telegram + звук/уведомление macOS).

## Быстрый старт (локально)

```bash
cd "чекер егэ"
pip3 install -r requirements.txt
cp config.example.json config.json   # затем впиши свои данные
python3 checker.py --once             # одна проверка (проверить настройки)
python3 checker.py                    # бесконечный цикл
```

`config.json`:

| Поле | Что это |
|---|---|
| `surname`, `name`, `patronymic` | ФИО как в регистрации на ЕГЭ |
| `reg_code` **или** `doc_number` | **Ровно одно**: код регистрации **или** номер документа без серии |
| `region` | Название (см. `regions.json`) или номер, напр. `"г. Москва"` |
| `captcha_solver` | `capmonster` (надёжно, нужен ключ) или `ocr` (бесплатно, но эту капчу читает плохо) |
| `capmonster_key` | ключ из ЛК CapMonster (если `captcha_solver=capmonster`) |
| `interval_seconds` | период проверки, сек (`900` = 15 минут) |
| `notify.telegram_bot_token` / `telegram_chat_id` | уведомления в Telegram (см. HOSTING.md) |

## Бесплатный хостинг 24/7 + Telegram

См. **[HOSTING.md](HOSTING.md)** — пошаговая настройка GitHub Actions (бесплатно) и Telegram-канала.
Коротко: код кладётся в публичный репозиторий, данные/ключи — в Secrets, GitHub сам запускает
проверку каждые 15 минут и шлёт результат в канал.

## Состояния (что пишет бот)

- **`Результатов пока нет («участник не найден»)`** — данные приняты, ждём публикации (нормально).
  Если так очень долго после официальной публикации — проверь ФИО/код/регион.
- **`Вход выполнен`** + таблица — участник найден; по каждому предмету видно балл или «Нет результата».
- **`🎉 ЕСТЬ НОВЫЕ РЕЗУЛЬТАТЫ!`** — по предмету появился балл; шлётся уведомление.
- **`ОШИБКА ДАННЫХ`** — сервис не принял данные (бот остановится, исправь config).

## Как это работает

1. `GET /api/captcha` → картинка (base64 JPEG) + `Token`.
2. Капча решается (CapMonster `ImageToTextTask` — **без** `numeric`, иначе теряется точность;
   принимаем только результат ровно из 6 цифр, иначе берём новую капчу).
3. `POST /api/participant/login` с MD5-хэшем ФИО, кодом/документом, регионом, капчей и токеном.
   Успех = HTTP **204** (ставится cookie `Participant`).
4. `GET /api/exam` → баллы по предметам.

`state.json` хранит **ID** уже увиденных результатов (только числа, без баллов) — чтобы не
уведомлять повторно и не светить баллы в публичном репозитории.

## Технические заметки

- Сайт не отдаёт промежуточный сертификат GlobalSign — поэтому в комплекте
  `globalsign-intermediate.pem`, и бот собирает CA-бандл `certifi + промежуточный` (иначе SSL-ошибка).
- Хэш ФИО: `md5(simplify(Фамилия+Имя+Отчество))`, где simplify = нижний регистр, убрать всё
  кроме букв, `ё→е`, `й→и` (проверено: совпадает с кодом сайта).
